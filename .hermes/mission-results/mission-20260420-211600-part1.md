## 3. 代码修复 (Bug Fixes)

### 3.1 测试 Bug 修复
**文件**: `tests/test_integration.py`
**问题**: `test_end_to_end()` 中 `pm` 变量在数据库初始化失败时未定义，导致 `UnboundLocalError`
**修复**: 添加 `pm = None` 初始化，并在使用前进行 `None` 检查

```python
# 修复前
pm = PersistentMemory(db_path=db_path, max_messages=100)
pm.add_user_message("搜索 python web framework")  # pm 可能未定义

# 修复后
pm = None  # Initialize to avoid UnboundLocalError
try:
    pm = PersistentMemory(...)
    ...
except Exception as e:
    ...
if pm is None:
    print("  工作流执行：✗ (数据库未就绪)")
else:
    pm.add_user_message("搜索 python web framework")
```

**问题**: 重复的 `python-dotenv` 条目，缺失 `aiosqlite`

- ✅ 清晰的模块化架构，关注点分离良好

- ✅ 对话管理器支持多轮对话和自动压缩

### 6.2 改进建议 (待未来迭代)
1. **SQLAlchemy 连接管理**: `PersistentMemory` 的异步引擎连接未显式关闭，GC 警告（非致命，但可优化）
2. **测试隔离**: 部分单元测试之间存在 ConfigManager 单例状态共享，可引入测试级 fixture 重置
3. **MCP 客户端**: `is_connected` 属性实现基于 `_session` 检查，但 `_session` 是父类内部属性，建议使用更稳定的连接状态判断
4. **代码复用**: `ResearcherAgent` 和 `AnalystAgent` 中 Studio 配置逻辑重复 (set_studio_config + _forward_to_studio)，可提取为共享模块
